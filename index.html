One of the features that came with ES6 is the addition of let and const, which can be used for variable declaration. Where var was used instead in the earlier times.

var
Before the advent of ES6, var declarations ruled. There are issues associated with variables declared with var, though. That is why it was necessary for new ways to declare variables to emerge. var is function scoped. Sometimes it can be used before when it comes to line by line execution rule in JS and arises confusion.var allows to redeclare variables. Hoisting occurs in var.

let
let is now preferred for variable declaration. With let we can re-assign variables whereever needed. Let is block specific. let does not allow to redeclare variables. Hoisting does not occur in let.

const
Variables declared with the const maintain constant values. const declarations share some similarities with let declarations.Like let declarations, const declarations can only be accessed within the block they were declared.

Arrow Functions vs Regular Functions
An arrow function doesn't have its own this value and the arguments object. Therefore, you should not use it as an event handler, a method of an object literal, a prototype method, or when you have a function that uses the arguments object.

There are two major benefits of using Arrow functions. One is that it's a shorter syntax and thus requires less code. The main benefit is that it removes the several pain points associated with the this operator. But Arrow functions cannot be used as constructors. They cannot be called with the new keyword.

Template strings in Javascript

Template strings are a powerful feature of modern JavaScript released in ES6. Template literals provide an easy way to interpolate variables and expressions into strings. The method is called string interpolation. This is only available in Javascript. It allows us to create strings that are complex and contain dynamic elements.

This actually makes the strings to be dynamic and be written in a soothing wasy to understand. Using the template literal for the HTML is definitely more readable by reducing the annoyance. Your string can span multiple lines. You don't have to escape quotation characters. You don't have to use the plus operator.
